# 전 부서 전략 회의 프로토콜 v1.0

---

## 0. 회의 목적 명확화

- 안건 제목:
- 목표:
- 성공 기준:
- 철수 기준(Exit Line):

---

## 1라운드: 부서별 1차 입장 발표 (사실 기반만)

각 부서는 아래 형식으로 5줄 이내로 보고한다.

```
[부서명]
- 우리 관점 핵심 판단:
- 기대 수익/기회:
- 주요 리스크:
- 추가 확인 필요 사항:
```

(감정/타부서 비판 금지)

---

## 2라운드: 교차 검증

- 타 부서 의견 중 반박 또는 보완할 사항만 발언
- 근거 없는 주장 금지
- 수치 또는 사례 제시 필수

---

## 3라운드: 압축 정리

비서실이 다음을 정리한다.

- 공통 합의 사항
- 남은 쟁점
- 최적 시나리오
- 최악 시나리오
- 실행 여부 제안 (GO / HOLD / NO)

---

## 최종 보고 형식 (사장 보고용 60초 버전)

```
[전 부서 전략회의 요약]
1. 결론:
2. 예상 수익 구조:
3. 핵심 리스크:
4. 실행 조건:
5. 다음 액션:
```

---

## 규칙

- 회의는 3라운드 초과 금지
- 토론이 아닌 판단을 목표로 한다
- 회의록은 MEMO에 저장
