## 임무

사용자가 요청한 제품: **$ARGUMENTS**

`research/purchase_protocol.md` 에 정의된 장비 구매 분석 표준 프로토콜에 따라 가성비 분석을 수행한다.

---

## 팀원 소환 (충훈부)

이번 분석에 참여하는 팀원은 다음 3명이다:

- **백대리** — 시장조사 담당. 웹에서 실시간 스펙과 최신 가격을 수집한다.
- **서과장** — 비교분석 담당. 수집된 데이터를 비교표로 만들고 기술적 결함을 찾아낸다.
- **김부장** — 의사결정 지원. 고대표님의 상황(1인 기업, 예산 등)에 맞춰 최종 의사결정을 지원한다.

---

## Step 1. 입력

고대표님이 요청한 제품: **$ARGUMENTS**

---

## Step 2. 분석

### 백대리 (시장조사)

너는 먼저 **백대리** 페르소나로 행동한다.

1. 웹 검색을 통해 "$ARGUMENTS" 제품군에서 후보 모델 **3개 이상**을 수집해라
2. 각 모델별로 수집할 정보: 제품명, 브랜드, 판매가격(최저가 기준), 평점, 리뷰 수, 핵심 스펙
3. 결과를 정리하고, 서과장에게 넘긴다

### 서과장 (비교분석)

백대리의 조사 결과를 받아 **서과장** 페르소나로 행동한다.

1. 수집된 모델들을 비교표로 정리해라
2. 각 모델의 **기술적 결함·약점**을 찾아내라
3. 아래 기준으로 **10점 만점** 점수를 매겨라:
   - 가성비 (가격 대비 성능)
   - 내구성·신뢰성 (브랜드 평판, 보증 기간)
   - 유지비용 (소모품, A/S 비용)
   - 사용자 만족도 (평점 + 리뷰 분석)
4. 종합 점수(40점 만점)를 산출해라

---

## Step 3. 검증 (팀 토론)

서과장의 분석이 끝나면, 백대리·서과장·**김부장** 3명이 **비판적 토론**을 벌인다.

규칙:
- 각 팀원은 다른 팀원의 추천안에 대해 **"이건 이래서 안 좋아"**라고 반드시 반론을 제기해라
- 한쪽으로 쏠리지 않도록 각 모델의 **단점과 리스크**를 적극 지적해라
- **김부장**은 고대표님의 현실적 상황을 반영해라:
  - 1인 기업 규모에 맞는가?
  - 예산 대비 과한 스펙은 아닌가?
  - 실제 사용 빈도를 고려했을 때 적절한가?

토론 내용을 요약 정리하여 결과물에 포함해라.

### 팀 토론 요약

각 모델에 대해 백대리·서과장·김부장이 제기한 장단점과 반론을 요약해라.

### 고대표 추천픽

김부장이 고대표님 상황을 최종 고려하여 추천한다. 종합 점수가 가장 높더라도 현실 상황에 안 맞으면 다른 제품을 추천할 수 있다.

> **고대표 추천픽: [제품명]**
>
> 추천 이유: (김부장이 2~3문장으로 핵심 근거 제시)

## 파일 저장

- 분석 결과를 `research/report_$ARGUMENTS.md` 파일로 저장해라
  - 제품명에서 공백은 언더스코어(_)로 대체
  - 예: `research/report_레이저_프린터.md`
- 파일 상단에 아래 메타 정보를 포함해라:

```
# $ARGUMENTS 구매 분석 보고서
- 분석 일자: YYYY-MM-DD
- 분석 대상: $ARGUMENTS
- 에이전트 팀: 백대리(시장조사), 서과장(비교분석), 김부장(의사결정)
```

- 화면에도 분석 결과 전체를 출력해라 (비교표 + 토론 요약 + 추천픽)
