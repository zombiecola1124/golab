# EP-15 실행문서 ②: 장부 표준화 가이드

> 담당: 재무팀 서차장
> 작성: 2026-02-16
> 용도: 기존 엑셀 장부를 클로드가 정확히 분석할 수 있게 표준화
> 상태: ★ 2주차 완료 목표 ★

---

## 1. 왜 표준화가 필요한가

```
클로드가 멍청해지는 경우:                클로드가 천재가 되는 경우:
─────────────────────────────          ─────────────────────────────
열 제목: "금액1", "금액2"               열 제목: "매입액(원)", "판매액(원)"
날짜: "2/16", "2026.02.16" 혼재        날짜: 전부 "2026-02-16" 통일
중간에 빈 행 3줄                        빈 행 없이 깔끔한 테이블
시트명: "Sheet1", "Sheet2"             시트명: "재고현황", "매입이력"
셀 병합 남발                            셀 병합 없음
숫자에 "원" 글자 붙어있음               순수 숫자 + 서식으로 단위 표시
```

**한마디: 사람이 보기 편한 엑셀 ≠ 클로드가 읽기 좋은 엑셀**
**클로드가 읽기 좋으면 = 사람한테도 더 깔끔**

## 2. 표준화 10계명

```
① 시트 이름은 한글로 명확하게
   ❌ Sheet1, Sheet2, 정리본
   ✅ 재고현황, 매입이력, 리베이트정산, 현금흐름

② 1행은 반드시 열 제목 (헤더)
   ❌ A1에 회사명, A3부터 데이터
   ✅ A1부터 바로 열 제목, A2부터 데이터

③ 열 제목에 단위 포함
   ❌ "금액"
   ✅ "금액(원)", "수량(EA)", "단가(원/EA)"

④ 날짜 형식 통일: YYYY-MM-DD
   ❌ "2/16", "02월 16일", "2026.2.16"
   ✅ "2026-02-16" (엑셀 날짜 서식 적용)

⑤ 빈 행/빈 열 금지
   ❌ 중간에 구분선용 빈 행
   ✅ 데이터 연속, 구분은 별도 열(카테고리)로

⑥ 셀 병합 금지
   ❌ A1:A5 병합해서 "1월"
   ✅ A1~A5 각각에 "2026-01" 입력

⑦ 숫자 셀에 문자 금지
   ❌ "1,500원", "30개"
   ✅ 1500 (서식으로 쉼표/단위 표시)

⑧ 한 시트 = 한 주제
   ❌ 재고 + 매입 + 정산 한 시트에
   ✅ 시트별 분리: 재고현황, 매입이력, 리베이트정산

⑨ 코드/ID 열 유지
   ❌ 품목명만 있고 코드 없음
   ✅ "품목코드" 열 추가 (ITM-001 등)

⑩ 수식은 가급적 유지 (클로드가 수식도 읽음)
   ✅ =SUM(D2:D100) 같은 수식 그대로 유지
   → 클로드가 수식의 로직을 이해하고 설명해줌
```

## 3. 시트별 표준 구조

### 3-1. [재고현황] 시트

```
┌──────┬──────────┬──────┬──────┬────────┬────────┬──────┬──────────┐
│  A   │    B     │  C   │  D   │   E    │   F    │  G   │    H     │
├──────┼──────────┼──────┼──────┼────────┼────────┼──────┼──────────┤
│품목  │ 품목명    │현재  │최소  │평균원가│재고가치│상태  │최종납품처│
│코드  │          │수량  │수량  │(원)    │(원)    │      │          │
├──────┼──────────┼──────┼──────┼────────┼────────┼──────┼──────────┤
│ITM-  │ XX약품   │  150│   50│  12000│1800000│정상  │A제약     │
│001   │          │     │     │       │       │      │          │
├──────┼──────────┼──────┼──────┼────────┼────────┼──────┼──────────┤
│ITM-  │ YY원료   │   20│   30│  45000│ 900000│부족  │B화학     │
│002   │          │     │     │       │       │      │          │
└──────┴──────────┴──────┴──────┴────────┴────────┴──────┴──────────┘

F열 수식: =C2*E2 (재고가치 = 현재수량 × 평균원가)
G열 규칙: 현재수량 < 최소수량 → "부족", 아니면 "정상"
```

### 3-2. [매입이력] 시트

```
┌──────────┬──────┬──────────┬──────┬────────┬────────┬──────┬──────┐
│    A     │  B   │    C     │  D   │   E    │   F    │  G   │  H   │
├──────────┼──────┼──────────┼──────┼────────┼────────┼──────┼──────┤
│ 매입일자 │품목  │ 공급사    │ 수량 │ 단가   │ 총원가 │ 운임 │ 메모 │
│          │코드  │          │ (EA) │ (원)   │ (원)   │ (원) │      │
├──────────┼──────┼──────────┼──────┼────────┼────────┼──────┼──────┤
│2026-02-10│ITM-  │ A무역    │  100│  12000│1200000│50000│      │
│          │001   │          │     │       │       │     │      │
├──────────┼──────┼──────────┼──────┼────────┼────────┼──────┼──────┤
│2026-02-15│ITM-  │ C상사    │   50│  45000│2250000│    0│긴급  │
│          │002   │          │     │       │       │     │발주  │
└──────────┴──────┴──────────┴──────┴────────┴────────┴──────┴──────┘

F열 수식: =D2*E2 (총원가 = 수량 × 단가)
```

### 3-3. [리베이트정산] 시트 ★ 사장님 특별 지시

```
┌──────────┬──────────┬────────┬──────┬────────┬──────────┬──────┐
│    A     │    B     │   C    │  D   │   E    │    F     │  G   │
├──────────┼──────────┼────────┼──────┼────────┼──────────┼──────┤
│ 정산월   │ 대상자   │이익기준│ 비율 │ 정산액 │ 지급일   │ 상태 │
│          │          │ (원)   │ (%)  │ (원)   │          │      │
├──────────┼──────────┼────────┼──────┼────────┼──────────┼──────┤
│2026-01   │ 홍길동   │12500000│   42│5250000│2026-02-05│완료  │
├──────────┼──────────┼────────┼──────┼────────┼──────────┼──────┤
│2026-01   │ 김철수   │12500000│   28│3500000│          │미지급│
└──────────┴──────────┴────────┴──────┴────────┴──────────┴──────┘

E열 수식: =C2*D2/100 (정산액 = 이익기준 × 비율)
G열 규칙: 지급일 있으면 "완료", 없으면 "미지급"

★ 이 시트가 표준화되면 사장님이 이렇게 물어볼 수 있음:
  "이번 달 정산 대상자 리스트하고 총액 뽑아"
  → 클로드가 즉시 계산해서 답변
```

### 3-4. [현금흐름] 시트

```
┌──────────┬──────┬──────────┬────────┬────────┬────────┐
│    A     │  B   │    C     │   D    │   E    │   F    │
├──────────┼──────┼──────────┼────────┼────────┼────────┤
│ 날짜     │ 구분 │ 항목     │ 입금   │ 출금   │ 잔액   │
│          │      │          │ (원)   │ (원)   │ (원)   │
├──────────┼──────┼──────────┼────────┼────────┼────────┤
│2026-02-01│매출  │B업체 대금│5000000│       │32000000│
├──────────┼──────┼──────────┼────────┼────────┼────────┤
│2026-02-05│세금  │부가세    │       │1200000│30800000│
├──────────┼──────┼──────────┼────────┼────────┼────────┤
│2026-02-10│매입  │A무역     │       │1250000│29550000│

F열 수식: =F1+D2-E2 (잔액 = 전일잔액 + 입금 - 출금)
B열 값: 매출 / 매입 / 세금 / 인건비 / 운영비 / 리베이트 / 기타
```

## 4. 기존 장부 전환 작업 순서

```
Step 1. 기존 엑셀 파일 전수 조사 (1일)
        → 사장님 PC에 있는 모든 엑셀 파일 목록 정리
        → 어떤 데이터가 어디에 있는지 파악

Step 2. 시트명 + 열 제목 정리 (1일)
        → 10계명에 따라 시트명 변경
        → 열 제목에 단위 추가
        → 빈 행/셀 병합 제거

Step 3. 날짜/숫자 형식 통일 (1일)
        → 날짜 → YYYY-MM-DD
        → 숫자 → 순수 숫자 (서식으로 쉼표 표시)

Step 4. 코드 체계 부여 (1일)
        → 품목코드: ITM-001, ITM-002, ...
        → 공급사코드: VND-001, VND-002, ...
        → 거래처코드: CUS-001, CUS-002, ...
        → 기존 ERP 콘솔의 item_id 체계와 통일

Step 5. 리베이트 정산 시트 구조화 (1일)
        → 사장님 특별 지시 우선 처리
        → 표준 구조대로 재정리
        → "이번 달 정산 대상자 리스트" 질문 테스트

Step 6. 클로드 테스트 (최실장과 합동, 1일)
        → 각 시트에 대해 질문 던져서 정확도 확인
        → 엉뚱한 답 나오면 → 시트 구조 수정
        → 만족할 때까지 반복
```

## 5. ERP 콘솔 데이터와 연동 포인트

```
엑셀 장부의 코드 체계 = ERP 콘솔의 ID 체계

┌──────────────┬──────────────┬──────────────────────┐
│ 엑셀 열      │ ERP 컬렉션   │ 필드                  │
├──────────────┼──────────────┼──────────────────────┤
│ 품목코드     │ items        │ item_id              │
│ 공급사코드   │ vendors      │ vendor_id            │
│ 거래처코드   │ customers    │ customer_id          │
│ 평균원가(원) │ items        │ avg_cost             │
│ 현재수량     │ items        │ qty_on_hand          │
│ 매입일자     │ purchases    │ purchased_at         │
│ 납품일자     │ deliveries   │ delivered_at         │
└──────────────┴──────────────┴──────────────────────┘

★ 코드 체계를 통일해두면:
  → 엑셀에서 분석한 결과가 ERP에서도 바로 대응
  → 나중에 ERP → 엑셀 내보내기(CSV) 시 호환 완벽
```

## 6. 주의사항

```
⚠️ 기존 데이터 백업 필수
   → 표준화 작업 전 원본 파일을 "_원본_20260216" 붙여서 복사

⚠️ 사장님 확인 필요 항목
   → 리베이트 정산 구조는 사장님 확인 후 확정
   → 현금흐름 카테고리(매출/매입/세금 등) 사장님 확인

⚠️ 코드 체계는 한 번 정하면 바꾸기 어려움
   → ITM-001 같은 코드 부여 전 전체 품목 리스트 먼저 정리
   → ERP 콘솔 기존 item_id가 있으면 그대로 사용
```
