## 2. 사장님 요청 에피소드 (전문 기록)

### EP-01. 전체 업무 흐름 확인

사업 흐름이 단순하지 않고 얽혀 있다는 문제 제기.
→ 정리 결과, 핵심 흐름은 하나:

```
견적 받고 → 수입(매입)하고 → 납품하고 → 돈 받고(정산)
```

각 단계가 별도 기록되면 "이 건 어디까지 진행됐나"가 보이는 구조.

### EP-02. 업체관리 강화 요청

**현재 문제:** vendors/customers 마스터가 필드 빈약 (이름/연락처/메모뿐)

**요청 내용:**
- 업체 누르면 해당 업체와의 **전체 히스토리**가 나와야 함
  - 견적 요청 받은 것, 보낸 것
  - 매입한 것, 납품한 것
- 업체명 / 부서명 / 담당자 / 연락처 / 메일주소
- 기존에 견적서 작성하며 정리해둔 업체 목록이 있음 (별도 관리 중)

**UI 방향:** 리스트(요약) → 클릭 → 상세(히스토리 쭉) 패턴 동의

### EP-03. 견적 관리 (신규 영역)

**요청 내용:**
- **보낸 견적:** 언제, 누구한테, 무슨 내용으로, 얼마에, 왜 보냈는지
- **받은 견적:** 어디서, 뭘, 얼마에 받았는지
- 견적이 나중에 매입/납품으로 연결되면 흐름 추적 가능

**비고:** 기존에 견적 송부 업체들은 업체명/부서명/담당자/연락처/메일/견적내용 별도 정리 중

### EP-04. 가격표 리스트

**요청 내용:**
- "이거 얼마지?" 하면 딱 나오는 가격표
- 내가 취급하는 품목 가격
- 업체에서 견적 받았던 것도 정리해서 볼 수 있으면 좋겠음
- **가격 이력:** 언제 기준 가격인지, 과거 이 금액 → 지금 이 금액 추적

### EP-05. 가격 추적 워치리스트 (신규 영역)

**요청 내용:**
- 관심 있거나 타겟으로 하는 아이템 가격 추적
- **최고가 / 최저가 / 평균가** 확인
- "지금 싸다 → 재고로 쌓아서 팔자" 판단용
- 매입 타이밍 의사결정 지원 도구

### EP-06. 배포 방식 — PC 소프트웨어처럼 쓰고 싶다

**사장님 질문:**
- 메인으로 내 PC에서 소프트웨어처럼 쓰길 원함
- 웹 vs 설치형 vs 구글드라이브 중 뭐가 맞는지

**→ 개발팀 답변:** PWA(Progressive Web App) 추천
- 웹인데 바탕화면 아이콘으로 앱처럼 실행
- 기존 설계에서 파일 2개만 추가 (manifest.json + sw.js)
- 개발 난이도 거의 안 바뀜

**→ 재무팀 답변:** 비용 0원 구조 확인
- Firebase Spark 무료 한도: 1인 사업자 기준 충분
- 시중 ERP 연 60~360만원 대비 무료
- 향후 DB 교체 대비 db.js 추상화 이미 설계됨

**→ 보안팀 답변:**
- v1 익명인증 → 브라우저 바뀌면 데이터 유실 위험
- v1.5 이메일/구글 로그인 일정 미루면 안 됨
- Firestore Security Rules 필수 설정
- 공유 PC 사용 시 로그아웃 처리 필요

**→ 운영팀 답변:**
- 환율 수동 입력 불편 → 자동 환율 v1.5에 고려
- 견적서 PDF 출력 → 메일 보내는 흐름 끊기면 안 씀
- 업체 연락처 폰에서 바로 전화/문자 가능해야 함 (tel: 링크)

### EP-07. 법인 vs 개인사업자 시뮬레이션 (재무팀 업무)

**사장님 지시:**
- 법인 전환 시 vs 개인사업자 유지 시 비교 시뮬레이션 필요
- 재무팀이 돌려줘야 할 항목:

**시뮬레이션 예상 비교 항목:**
- **세금 구조:** 종합소득세(개인, 누진 6~45%) vs 법인세(법인, 구간별 9~24%) + 대표 급여 소득세
- **4대보험:** 개인(지역가입자) vs 법인(직장가입자, 회사 부담분 발생)
- **부가세:** 구조 동일하나 매입세액공제 범위 차이
- **비용 인정 범위:** 법인이 넓음 (차량, 접대비 한도, 복리후생 등)
- **자금 인출:** 개인(자유) vs 법인(급여/배당/가지급금 — 세금 이중과세 이슈)
- **대외 신용도:** 법인이 유리 (입찰, 거래처 신뢰, 대출)
- **설립/유지 비용:** 법인 설립비, 법무사 비용, 세무사 기장료 차이
- **엑시트 전략:** 사업 매각/양도 시 개인 vs 법인 차이

**재무팀 상태:** 시뮬레이션 완료 (006-a_SIM_법인vs개인_20260214.md)
**재무팀 권고:** 법인 설립 추천 (288억 거래 무한책임 리스크 + 성장 시 절세 효과)
**사장님 판단:** 대기 중

### EP-08. 일본 전시회 리서치 + 시장 개척 구상

**사장님 브리핑:**
- 작년 일본 전시회 출장 11~14일, 업체 부스 사진 촬영해옴
- "일단 저질렀는데 어디서 시작하고 끝낼지 막막"
- 업체 정리 / 시장 분석 / 사업 연결 / 블로그 기록 희망
- 전략: "일본의 폐쇄적 시장을 깨부셔서 써먹겠다"

**전 팀 회의 완료** → 상세: 006-b_EP08_일본전시회리서치_20260214.md

**비서실 평가:** 전략적 타당성 ★★★★☆ (4/5)
**재무팀 평가:** 마진율 개선의 가장 현실적 경로. 투자 회수 가능성 높음.
**개발팀 방향:** 별도 프로젝트로 구축, ERP와 vendor_id 연결 (CLAUDE.md 원칙 유지)
**사장님 결정 대기:** 사진 정리 착수 여부, 블로그 플랫폼, 업체 접촉 방식, 우선순위
