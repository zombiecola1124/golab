# 스킬 운영 정책 (Skill Operation Policy)

> 최종 갱신: 2026-02-13
> 승인: 사장
> 관리: 비서실(최실장)

---

## 1. 운영 원칙

1. **ERP 핵심 영역에는 자동 스킬 설치를 금지한다.**
2. 외부 스킬은 반드시 **화이트리스트 방식**으로만 사용한다.
3. 스킬은 **"속도 향상 → 수익 창출"이 명확할 때만** 도입한다.
4. 모든 스킬 사용은 **비용 대비 효과를 기록**한다.

---

## 2. 사용 영역 구분

### 🟥 ERP 핵심 영역 (금지 영역) — 내부 고정 로직만 사용

| 영역 | 설명 |
|------|------|
| 재고 수량 계산 | qty_on_hand 연산 |
| 평균원가 계산 | 이동평균법 (calcWeightedAvgCost) |
| 세금계산서 금액 처리 | supply_amount / vat_amount 계산 |
| 미수금 계산 | paid vs unpaid 정산 |

> 이 영역은 외부 AI 스킬/API 개입 금지. js/app.js 내부 함수로만 처리.

### 🟩 속도 무기화 영역 (적극 활용)

| 영역 | 활용 예시 |
|------|----------|
| 신규 고객 제안서 제작 | 템플릿 자동 생성 |
| 기술 비교 보고서 작성 | 경쟁 제품 분석 |
| 시장 조사 정리 | 트렌드 요약 |
| 시각화 자료 제작 | 차트/그래프 자동 생성 |
| 경쟁사 분석 | 가격/스펙 비교표 |

## 4. 스킬 도입 절차 (검증 체크리스트)

신규 스킬 도입 시 **3단계 검증 필수 통과**:

### ✔ 1단계: 보안 점검

- [ ] 외부 API 키 필요 여부 확인
- [ ] ERP DB 접근 여부 차단 확인
- [ ] 데이터 외부 전송 여부 확인

### ✔ 2단계: 비용 점검

- [ ] API 사용 비용 발생 여부
- [ ] 무료 대안 존재 여부
- [ ] 예상 월 사용 비용 계산

### ✔ 3단계: 수익성 점검

- [ ] 작업 시간 몇 % 단축?
- [ ] 그 단축으로 매출 증대 가능한가?
- [ ] 실제 돈으로 환산 가능?

**3단계 모두 통과한 스킬만 사용 승인.**

## 5. Self-Healing 전략

- ❌ 자동 우회 금지
- ✅ 실패 시 로그 기록
- ✅ 수동 대체 스킬 호출
- ✅ 재시도는 **1회만**
- ❌ 정책 위반 가능성 있는 우회는 하지 않음
