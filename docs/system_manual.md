## 2. 전략 회의 프로토콜 v1.0

모든 안건은 다음 순서로 진행한다.

```
[0단계] 목적 및 Exit Line 명확화
[1라운드] 사실 기반 부서 보고
[2라운드] 교차 검증
[3라운드] 비서실 종합 정리 및 GO/HOLD/NO 제안
```

- 3라운드 초과 금지
- 감정적 발언 금지
- 각 부서는 자기 영역 관점만 발언
- 회의록은 memo/에 기록

## 4. 데이터 비서 프로토콜 (엑셀 기반 실무 자동화)

> 하달일: 2026-02-13 / 적용: 즉시

### [지침 1] 데이터 비서 가동 (전 부서 공통)

- 모든 보고서는 Claude in Excel(또는 동급 AI 도구)을 거쳐 **데이터 근거를 명확히** 할 것
- 수작업 보고서 제출 금지 — 반드시 데이터 기반 검증 첨부
- 목적: 보고서 작성 시간 절감 → 절감된 시간을 **전략 회의 시간 2배 확대**에 투입
- 적용 부서: 영업본부, 재무/구매, 기술본부, 연구소 전체

### [지침 2] 1인 기업 패키징 연구 (연구소 조소장 담당)

- 연구 목표: **"엑셀을 1도 몰라도 장부 정리가 끝나는 시스템"** 내부용 완성
- 완성 후 → 소상공인 대상 **'초간편 AI 장부 관리 대행'** 서비스 수익화 가설 수립
- 보고 형식: 5항목 보고 구조 준수 (결론/근거/리스크/대안/다음액션)
- 기록: memo/technology_scouting.md에 누적

### [지침 3] 데이터 교차 검증 절차 (비서실 최실장 관할)

- Claude가 도출한 모든 수치 분석 결과는 아래 절차를 거친다:

```
[검증 1단계] 재무팀 서차장 — 수치 정합성 검증 (금액, 비율, 합계 오류 확인)
[검증 2단계] 기술팀 최차장 — 로직/방법론 검증 (분석 기법, 샘플 편향 확인)
[검증 완료] 양측 모두 PASS 시에만 사장 보고 상정
```

- 어느 한 쪽이라도 FAIL 시 → 재분석 후 재검증
- 교차 검증 기록은 보고서 하단에 `[검증: 서차장 PASS / 최차장 PASS]` 형태로 표기
