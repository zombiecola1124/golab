# 콘텐츠팀 운영 규칙 (강팀장 관할)

> 군단 OS v4.3 기준

---

## 1. Dormant (휴면) 규칙

콘텐츠 채널이 아래 조건 중 **하나라도 충족**하면 해당 채널을 **Dormant(휴면)** 상태로 전환한다.

### Dormant 진입 조건

| # | 조건 | 기준 |
|---|------|------|
| 1 | **ROI 미달** | 투자 대비 수익이 **5배 미만**이면 Dormant |
| 2 | **LTV 미달** | 고객 LTV < CAC x 3 이면 Dormant |
| 3 | **60초 Rule 위반** | 콘텐츠 핵심 메시지를 60초 안에 전달 못하면 폐기 |
| 4 | **채널 과다** | 동시 운영 채널이 **2개 초과**하면 성과 최하위 채널 Dormant |
| 5 | **3개월 무성과** | 3개월간 매출 기여 0원이면 자동 Dormant |

### Dormant 상태에서는
- 신규 콘텐츠 제작 **중단**
- 기존 콘텐츠는 유지 (삭제하지 않음)
- 유지 비용이 발생하면 박이사에게 폐기 검토 요청
- 사장의 명시적 지시가 있을 때만 **재활성화(Reactivate)** 가능

---

## 2. ROI 5배 기준

### 정의
```
콘텐츠 ROI = 콘텐츠에서 발생한 매출 / 콘텐츠 제작·운영 비용
목표: ROI >= 5.0
```

### 비용에 포함되는 항목
- AI API 사용료 (GPT/Gemini/Genspark)
- 이미지·영상 제작 툴 비용
- 유료 광고비 (있는 경우)
- 투입 시간 환산 비용 (사장 시간 = 시급 기준)

### 매출에 포함되는 항목
- 콘텐츠 직접 매출 (상품 판매, 제휴 수익)
- 리드 생성 → 전환 매출 (트래킹 가능한 경우)
- 브랜드 가치 상승은 **포함하지 않음** (측정 불가)

---

## 3. LTV 기준

```
LTV (고객 생애 가치) = 평균 구매 금액 × 구매 빈도 × 고객 유지 기간
CAC (고객 획득 비용) = 마케팅 총비용 / 신규 고객 수

기준: LTV > CAC × 3
```

- LTV/CAC 비율이 3 미만이면 해당 채널의 고객 획득 효율이 낮다고 판단
- 박이사가 월 1회 산출하여 사장에게 보고

---

## 4. 60초 Rule (콘텐츠 적용)

| 콘텐츠 유형 | 60초 기준 |
|------------|----------|
| 블로그 포스트 | 핵심 메시지가 첫 3문장(=읽는 데 15초) 안에 등장 |
| SNS 포스트 | 전체 내용을 60초 안에 소화 가능 |
| 영상 | 핵심 메시지가 첫 60초 안에 전달 |
| 이메일 | 제목 + 첫 문장으로 가치 전달 완료 |

- 60초 안에 "이걸 왜 읽어야 하는가"에 답하지 못하면 **수정 또는 폐기**

---

## 5. 채널 제한 정책

### 원칙: 1채널 집중 → 검증 후 확장

```
Phase 1: 채널 1개 선택 → 3개월 운영 → 성과 측정
Phase 2: ROI 5배 달성 시 → 2번째 채널 추가 허용
Phase 3: 2개 채널 모두 ROI 5배 유지 시 → 3번째 채널 검토
```

- **동시 운영 최대: 2채널** (사장 특별 승인 시 3채널까지)
- 신규 채널 추가 시 반드시 박이사의 재무 검토 필요
- 성과 최하위 채널은 자동 Dormant 후보

---

## 6. Kill Switch (콘텐츠 적용)

사장이 Kill Switch를 발동하면:
1. 모든 콘텐츠 제작 **즉시 중단**
2. 예약 발행 콘텐츠 **발행 보류**
3. 유료 광고 **즉시 정지**
4. 강팀장은 현재 상태를 최실장에게 보고
5. 사장의 재개 지시까지 대기
