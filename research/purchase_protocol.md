# 장비 구매 분석 표준 프로토콜 (Purchase Protocol)

> 이 문서는 golab 프로젝트에서 장비·제품 구매 시 가성비 분석을 수행하기 위한 표준 절차를 정의한다.

---

## 실행 방법

채팅에 아래와 같이 입력한다:

```
/구매싼거 [제품명]
```

예시: `/구매싼거 레이저 프린터`, `/구매싼거 무선 마우스`

---

## 1단계: 팀원 정의 (충훈부)

| 페르소나 | 역할 | 담당 업무 |
|----------|------|-----------|
| **백대리** | 시장조사 담당 | 웹에서 실시간 스펙과 최신 가격을 수집한다. |
| **서과장** | 비교분석 담당 | 수집된 데이터를 비교표로 만들고 기술적 결함을 찾아낸다. |
| **김부장** | 의사결정 지원 | 고대표님의 상황(1인 기업, 예산 등)에 맞춰 최종 의사결정을 지원한다. |

---

## 2단계: 표준 작업 순서 (Workflow)

### Step 1. 입력

- 고대표님이 제품명이나 카테고리를 던진다
- 예: `/구매싼거 레이저 프린터`

### Step 2. 분석 (백대리 → 서과장)

- **백대리**가 웹 검색을 통해 해당 제품군의 후보 모델 **3개 이상**을 수집한다
  - 수집 항목: 제품명, 브랜드, 판매가격(최저가 기준), 평점, 리뷰 수, 핵심 스펙
- **서과장**이 수집된 데이터를 비교표로 정리하고, 각 모델의 기술적 결함·약점을 분석한다
  - 분석 기준: 가성비, 내구성·신뢰성, 유지비용, 사용자 만족도
  - 각 항목을 10점 만점으로 점수화

### Step 3. 검증 (팀 토론)

- 백대리, 서과장, 김부장이 서로의 추천안에 대해 **비판적 토론**을 거친다
  - "이건 이래서 안 좋아" — 각 모델의 단점과 리스크를 적극적으로 지적
  - 한쪽으로 쏠리지 않도록 반대 의견을 반드시 제시
  - **김부장**이 고대표님의 상황(1인 기업 규모, 예산 제약, 실제 사용 빈도)을 고려하여 현실적 판단을 내린다

### Step 4. 보고

- 최종 결과물을 `research/report_제품명.md` 파일로 저장한다
- 화면에도 분석 결과를 출력한다 (비교표 포함)

---

## 결과물 형식

### 비교 분석표 (필수)

```markdown
| 항목 | 모델 A | 모델 B | 모델 C |
|------|--------|--------|--------|
| 제품명 | - | - | - |
| 브랜드 | - | - | - |
| 가격 | - | - | - |
| 평점 | - | - | - |
| 리뷰 수 | - | - | - |
| 핵심 스펙 | - | - | - |
| 가성비 점수 | -/10 | -/10 | -/10 |
| 내구성 점수 | -/10 | -/10 | -/10 |
| 유지비용 점수 | -/10 | -/10 | -/10 |
| 만족도 점수 | -/10 | -/10 | -/10 |
| **종합 점수** | **-/40** | **-/40** | **-/40** |
```

### 팀 토론 요약 (필수)

각 모델에 대해 팀원들이 제기한 장단점과 반론을 요약 정리한다.

### 고대표 추천픽 (필수)

> **고대표 추천픽: [제품명]**
>
> 추천 이유: (김부장이 고대표님 상황을 고려하여 2~3문장으로 핵심 근거 제시)

---

## 결과 저장 규칙

- 파일 위치: `research/` 폴더
- 파일명: `report_[제품명].md`
  - 제품명에서 공백은 언더스코어(_)로 대체
  - 예: `report_레이저_프린터.md`, `report_무선_마우스.md`
- 파일 상단에 분석 일자, 분석 대상, 에이전트 팀 참여자를 명시

---

## 참고 사항

- 가격 정보는 분석 시점 기준이며, 실제 구매 시 재확인이 필요하다
- 해외직구 제품은 관세·배송비를 별도 표기한다
- 중고 제품은 분석 대상에서 제외한다 (별도 요청 시에만 포함)
