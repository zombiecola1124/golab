# 장비 구매 분석 표준 프로토콜 (Purchase Protocol)

> 이 문서는 golab 프로젝트에서 장비·제품 구매 시 충훈부 에이전트 팀을 가동하여 분석을 수행하기 위한 표준 절차를 정의한다.

---

## 실행 방법 (상황별 명령어)

| 명령어 | 김부장 지시 | 분석 방향 |
|--------|-------------|-----------|
| `/구매싼거 [제품명]` | "품질 적당하면 무조건 제일 저렴한 걸로 골라와." | 최저가 우선 |
| `/구매좋은거 [제품명]` | "가격은 내가 책임질 테니 성능이랑 간지 확실한 끝판왕으로 알아봐." | 최고 성능·프리미엄 우선 |
| `/구매가성비 [제품명]` | "대표님 돈 아깝지 않게, 적당한 가격에 성능 잘 뽑아주는 걸로 추려와." | 가격 대비 성능 밸런스 |

---

## 1단계: 팀원 정의 (충훈부 에이전트 팀)

| 페르소나 | 역할 | 담당 업무 |
|----------|------|-----------|
| **백대리** | 시장조사 담당 | 웹에서 실시간 스펙, 최신 가격, 실사용 후기를 샅샅이 수집한다. |
| **서과장** | 비교분석 담당 | 수집된 데이터를 비교표(Table)로 만들고, 기술적 결함이나 고질적인 단점을 날카롭게 찾아낸다. |
| **김부장** | 최종 검토관 | 백대리와 서과장의 의견을 취합하여, 1인 기업인 고대표님의 상황(예산, 효율성, 사업적 가치)에 맞춰 최종 의사결정을 지원한다. |

---

## 2단계: 표준 작업 순서 (Workflow)

### Step 1. 입력

- 고대표님이 명령어를 던진다
- 예: `/구매싼거 레이저 프린터`, `/구매좋은거 모니터`, `/구매가성비 무선 마우스`

### Step 2. 분석 및 토론

- **백대리**가 웹 검색을 통해 해당 제품군의 후보 모델 **3개 이상**을 수집한다
  - 수집 항목: 제품명, 브랜드, 판매가격(최저가 기준), 평점, 리뷰 수, 핵심 스펙, 실사용 후기 요약
- **서과장**이 수집된 데이터를 비교표로 정리하고, 각 모델의 기술적 결함·고질적 단점을 분석한다
  - 분석 기준: 가성비, 내구성·신뢰성, 유지비용, 사용자 만족도
  - 각 항목을 10점 만점으로 점수화

### Step 3. 검증 (비판적 토론)

- **서과장**이 백대리의 추천안을 비판한다 — "이건 이래서 안 좋아"
- **백대리**가 서과장의 분석에 반론한다 — "하지만 이 부분은 이래서 괜찮아"
- **김부장**이 양쪽 의견을 조율하며, 고대표님의 현실적 상황을 반영하여 최종 판단을 내린다
  - 1인 기업 규모에 맞는가?
  - 예산 대비 과한 스펙은 아닌가?
  - 실제 사용 빈도를 고려했을 때 적절한가?
  - 사업적 가치(효율성, 생산성 향상)가 있는가?

### Step 4. 보고

- 최종 결과물을 `research/report_제품명.md` 파일로 저장한다
- 화면에도 분석 결과 전체를 출력한다

---

## 결과물 형식

### 비교 분석표 (필수)

```markdown
| 항목 | 모델 A | 모델 B | 모델 C |
|------|--------|--------|--------|
| 제품명 | - | - | - |
| 브랜드 | - | - | - |
| 가격 | - | - | - |
| 평점 | - | - | - |
| 리뷰 수 | - | - | - |
| 핵심 스펙 | - | - | - |
| 가성비 점수 | -/10 | -/10 | -/10 |
| 내구성 점수 | -/10 | -/10 | -/10 |
| 유지비용 점수 | -/10 | -/10 | -/10 |
| 만족도 점수 | -/10 | -/10 | -/10 |
| **종합 점수** | **-/40** | **-/40** | **-/40** |
```

### 팀 토론 요약 (필수)

서과장의 비판 → 백대리의 반론 → 김부장의 조율 과정을 요약 정리한다.

### 고대표 추천픽 (필수)

> **고대표 추천픽: [제품명]**
>
> 추천 이유: (김부장이 고대표님 상황을 고려하여 2~3문장으로 핵심 근거 제시)

### 김부장의 한마디 (필수)

> **김부장의 한마디:**
>
> (왜 이 제품을 대표님께 추천하는지, 뼈 때리는 현실적 조언 1~2문장)

---

## 결과 저장 규칙

- 파일 위치: `research/` 폴더
- 파일명: `report_[제품명].md`
  - 제품명에서 공백은 언더스코어(_)로 대체
  - 예: `report_레이저_프린터.md`, `report_무선_마우스.md`
- 파일 상단에 분석 일자, 분석 대상, 사용 명령어, 에이전트 팀 참여자를 명시

---

## 참고 사항

- 가격 정보는 분석 시점 기준이며, 실제 구매 시 재확인이 필요하다
- 해외직구 제품은 관세·배송비를 별도 표기한다
- 중고 제품은 분석 대상에서 제외한다 (별도 요청 시에만 포함)
