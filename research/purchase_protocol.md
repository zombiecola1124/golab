# 장비 구매 분석 표준 프로토콜 (Purchase Protocol)

> 이 문서는 golab 프로젝트에서 장비·제품 구매 시 가성비 분석을 수행하기 위한 표준 절차를 정의한다.

---

## 실행 방법

채팅에 아래와 같이 입력한다:

```
/구매싼거 [제품명]
```

예시: `/구매싼거 레이저 프린터`, `/구매싼거 무선 마우스`

---

## 에이전트 팀 구성

| 페르소나 | 역할 | 담당 업무 |
|----------|------|-----------|
| **백대리** (시장조사 담당) | 리서처 | 온라인 쇼핑몰·리뷰 사이트를 조사하여 후보 제품 3개를 선별한다. 평점·가격·판매량 데이터를 수집한다. |
| **서과장** (비용분석 담당) | 분석가 | 백대리가 수집한 데이터를 기반으로 가성비를 정량 분석한다. TCO(총소유비용), 내구성, A/S 등을 종합 평가한다. |

---

## 분석 워크플로우

### 1단계: 시장조사 (백대리)

- 해당 제품군에서 **평점 4.0 이상**, **리뷰 100건 이상**인 제품을 우선 필터링
- 가격대별로 **저가 / 중가 / 고가** 3개 모델을 선별
- 수집 항목:
  - 제품명 / 브랜드
  - 판매가격 (최저가 기준)
  - 평점 (5점 만점)
  - 리뷰 수
  - 주요 스펙 (제품군별 핵심 사양)

### 2단계: 비용분석 (서과장)

- 백대리가 선별한 3개 모델에 대해 **가성비 점수**를 산출
- 분석 기준:
  - **가격 대비 성능** (스펙/가격 비율)
  - **내구성·신뢰성** (브랜드 평판, 보증 기간)
  - **유지비용** (소모품, A/S 비용)
  - **사용자 만족도** (평점 + 리뷰 키워드 분석)
- 각 항목을 10점 만점으로 점수화

### 3단계: 최종 보고

- 비교 분석 결과를 **표(Table) 형식**으로 정리
- 마지막에 **'고대표 추천픽'**을 선정하여 추천 이유와 함께 제안

---

## 결과물 형식

### 비교 분석표 (필수)

```markdown
| 항목 | 모델 A | 모델 B | 모델 C |
|------|--------|--------|--------|
| 제품명 | - | - | - |
| 브랜드 | - | - | - |
| 가격 | - | - | - |
| 평점 | - | - | - |
| 리뷰 수 | - | - | - |
| 핵심 스펙 | - | - | - |
| 가성비 점수 | -/10 | -/10 | -/10 |
| 내구성 점수 | -/10 | -/10 | -/10 |
| 유지비용 점수 | -/10 | -/10 | -/10 |
| 만족도 점수 | -/10 | -/10 | -/10 |
| **종합 점수** | **-/40** | **-/40** | **-/40** |
```

### 고대표 추천픽 (필수)

> **고대표 추천픽: [제품명]**
>
> 추천 이유: (2~3문장으로 핵심 근거 제시)

---

## 결과 저장 규칙

- 파일 위치: `research/` 폴더
- 파일명: `purchase_[제품군]_[YYYYMMDD].md`
  - 예: `purchase_레이저프린터_20260211.md`
- 파일 상단에 분석 일자, 분석 대상, 에이전트 팀 참여자를 명시

---

## 참고 사항

- 가격 정보는 분석 시점 기준이며, 실제 구매 시 재확인이 필요하다
- 해외직구 제품은 관세·배송비를 별도 표기한다
- 중고 제품은 분석 대상에서 제외한다 (별도 요청 시에만 포함)
